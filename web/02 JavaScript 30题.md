#### 38. 什么是 XSS？如何防范？

#### 45. 什么是垃圾回收机制？

JavaScript 的垃圾回收机制是 JS 引擎自动管理内存的机制。程序运行时会不断创建对象、数组、函数等数据，当这些数据不再被使用时，垃圾回收器会在合适的时机释放它们占用的内存。现代 JavaScript 引擎主要通过标记清除法判断对象是否可回收：从全局对象、调用栈变量等根对象出发，能访问到的对象认为仍然可用，访问不到的对象认为是垃圾并回收。

JS 自动回收不再使用的内存 。
常见算法：

- 标记清除：标记所有变量，然后清除环境中无法访问的变量。
- 引用计数：统计引用次数，为 0 时回收（但循环引用会出问题）。



#### 46. 什么是内存泄漏？常见原因？

本该被回收的数据，因为仍然被引用，导致无法释放。

不再使用的内存没有被释放，导致内存占用越来越多。
常见原因：

- 全局变量。
- 未清理的定时器、事件监听函数
- 闭包。
- DOM 引用未清理。



#### 47. 什么是模块化？CommonJS 和 ES6 模块区别？

模块化是把代码拆分成一个个独立模块，每个模块只暴露需要对外使用的内容，其他文件通过导入来使用。它可以提高代码复用性、可维护性，也能避免全局变量污染。

CommonJS 是 Node.js 常用的模块规范，使用 `require` 导入，使用 `module.exports` 或 `exports` 导出，它是运行时加载。ES6 模块是 JavaScript 官方模块规范，使用 `import` 和 `export`，它是编译时静态加载，支持静态分析和 Tree Shaking。两者的主要区别在于语法不同、加载时机不同、导出值特性不同：CommonJS 更偏运行时，ES6 模块更适合现代前端工程化。

- CommonJS：Node.js 标准，`require` / `module.exports`，同步加载，运行时加载。
- ES6 模块：`import` / `export`，编译时加载，静态分析。

#### 50. `null` 和 `undefined` 区别？

- `undefined`：表示“缺少值”，变量已声明但未赋值。
- `null`：表示“空值”，显式赋值为 `null`。

`undefined` 表示变量已经声明但没有赋值，也常出现在对象属性不存在、函数没有返回值、参数没有传的情况。`null` 表示一个人为设置的空值，通常用来表示当前没有对象或数据。它们使用 `==` 比较是 `true`，但使用 `===` 比较是 `false`，因为类型不同。另外 `typeof undefined` 是 `'undefined'`，而 `typeof null` 是 `'object'`，这是 JavaScript 的历史遗留问题。

所以 `null == undefined` 是一条特殊规则，不是因为它们都转换成了 `0` 或 `false`。

#### 58. 什么是函数柯里化？

函数柯里化是把一个多参数函数转换成多个单参数或部分参数函数的过程。它可以让函数分批接收参数，等参数收集完整后再执行。柯里化常用于参数复用、延迟执行和函数组合。例如 `add(1, 2, 3)` 可以转换成 `add(1)(2)(3)`。

```
function curry(fn) {
  return function curried(...args) {
    if (args.length >= fn.length) {
      return fn(...args)
    }

    return function (...nextArgs) {
      return curried(...args, ...nextArgs)
    }
  }
}
```

#### 60. 什么是尾调用优化？

尾调用是指一个函数的最后一步是调用另一个函数。尾调用优化是指引擎可以复用当前函数的调用栈，不再新增调用栈，从而减少内存占用，避免深层递归导致栈溢出。

常见场景是尾递归，比如把 `return n + fn(n - 1)` 改成 `return fn(n - 1, total + n)`。不过 JavaScript 中尾调用优化支持并不稳定，实际开发中深递归通常更推荐改成循环
