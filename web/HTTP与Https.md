HTTP 和 HTTPS 的核心区别：

```txt
HTTPS = HTTP + TLS/SSL 加密层
```

也就是说，HTTPS 不是一种完全新的协议，而是在 HTTP 和 TCP 之间加了一层安全机制。

------

# 1. 安全性不同

## HTTP

HTTP 是明文传输。

例如你提交：

```txt
用户名：zhangsan
密码：123456
```

如果使用 HTTP，中间人理论上可以直接看到传输内容。

## HTTPS

HTTPS 会先进行 TLS 握手，建立加密通道，然后再传输 HTTP 数据。

即使别人截获了数据，也很难直接看懂内容。

面试可以说：

> HTTP 明文传输，数据容易被窃听、篡改；HTTPS 通过 TLS 对数据加密，并能验证服务器身份，安全性更高。

------

# 2. 端口不同

默认端口：

```txt
HTTP：80
HTTPS：443
```

例如：

```txt
http://example.com
https://example.com
```

浏览器默认会分别走 80 和 443 端口。

------

# 3. URL 前缀不同

HTTP：

```txt
http://
```

HTTPS：

```txt
https://
```

现在大多数正式网站都会使用 HTTPS。

------

# 4. 是否需要证书不同

HTTP 不需要证书。

HTTPS 需要服务器配置 SSL/TLS 证书。

证书的作用是：

```txt
证明当前服务器确实是你要访问的网站
防止中间人伪造服务器
参与加密通信过程
```

例如你访问银行网站时，浏览器会检查证书是否合法、是否过期、域名是否匹配。

------

# 5. 传输过程不同

HTTP 流程大致是：

```txt
TCP 三次握手
→ 发送 HTTP 请求
→ 返回 HTTP 响应
```

HTTPS 流程大致是：

```txt
TCP 三次握手
→ TLS 握手
→ 发送加密后的 HTTP 请求
→ 返回加密后的 HTTP 响应
```

所以 HTTPS 多了一步 TLS 握手。

------

# 6. 性能开销不同

HTTPS 因为多了 TLS 握手和加解密过程，理论上比 HTTP 多一点开销。

但现在浏览器、服务器和协议都有优化，比如：

```txt
TLS 1.3
会话复用
HTTP/2
硬件加速
```

所以实际性能差距通常不大。

------

# 面试回答版本

可以这样说：

> HTTP 和 HTTPS 最大的区别是安全性。HTTP 是明文传输，数据可能被窃听或篡改；HTTPS 是在 HTTP 基础上加入 TLS/SSL 加密层，可以实现数据加密、身份认证和完整性校验。HTTP 默认端口是 80，HTTPS 默认端口是 443。HTTPS 需要配置证书，并且在发送 HTTP 请求前会先进行 TLS 握手，因此比 HTTP 多一点性能开销，但现在优化后影响通常不大。总体来说，正式网站一般都应该使用 HTTPS。

------

你可以记这句话：

```txt
HTTP 是明文传输，HTTPS = HTTP + TLS，解决加密、认证、防篡改问题。
```
