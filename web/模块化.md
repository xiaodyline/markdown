# 模块导出

## 一、什么是模块导出

在 JavaScript 和 TypeScript 中，一个文件通常可以看作一个模块。
模块导出的作用，是把当前文件中的变量、函数、类、对象等内容暴露出去，供其他文件导入使用。

常见的导出方式有两种：

1. `export`
2. `export default`

------

## 二、export

`export` 叫做命名导出，也可以理解为分别导出。

### 1. 基本写法

```ts
export const name = 'Tom';
export const age = 20;

export function sayHello() {
  console.log('hello');
}
```

### 2. 导入方式

使用命名导出时，导入要加 `{}`。

```ts
import { name, age, sayHello } from './test';
```

### 3. 起别名

如果导入时想换名字，可以使用 `as`。

```ts
import { name as userName } from './test';
```

### 4. 特点

- 一个文件中可以有多个 `export`
- 导入时一般需要使用 `{}``
- 导入的名字要和导出的名字对应
- 更适合导出多个工具函数、多个变量、多个类型

------

## 三、export default

`export default` 叫做默认导出。

一个模块可以有很多命名导出，但通常只能有一个默认导出。

### 1. 基本写法

```ts
const user = {
  name: 'Tom',
  age: 20,
};

export default user;
```

也可以直接写：

```ts
export default function App() {
  return <div>App</div>;
}
```

### 2. 导入方式

默认导出在导入时不需要 `{}`。

```ts
import user from './test';
```

### 3. 导入时名字可自定义

```ts
import abc from './test';
```

这里的 `abc` 可以随便写，因为默认导出不要求导入名和原名一致。

### 4. 特点

- 一个文件通常只能有一个 `export default`
- 导入时不需要 `{}``
- 导入名可以自定义
- 更适合导出当前文件最核心的内容

------

## 四、两者的区别

| 对比项              | export     | export default |
| ------------------- | ---------- | -------------- |
| 含义                | 命名导出   | 默认导出       |
| 一个文件可导出数量  | 可以多个   | 通常一个       |
| 导入时是否需要 `{}` | 需要       | 不需要         |
| 导入名是否必须对应  | 一般要对应 | 可以自定义     |

------

## 五、代码对比

### 1. 使用 export

```ts
export const a = 1;
export const b = 2;
```

导入：

```ts
import { a, b } from './test';
```

### 2. 使用 export default

```ts
const a = 1;
export default a;
```

导入：

```ts
import a from './test';
```

------

## 六、可以同时使用

一个模块中可以同时写命名导出和默认导出。

### 1. 导出

```ts
export const age = 20;

const user = {
  name: 'Tom',
};

export default user;
```

### 2. 导入

```ts
import user, { age } from './test';
```

这里：

- `user` 是默认导入
- `age` 是命名导入

------

## 七、默认导出对象后的解构问题

如果 `export default` 导出的是一个对象，可以先导入，再对对象进行解构。

### 1. 导出对象

```ts
const user = {
  name: 'Tom',
  age: 20,
};

export default user;
```

### 2. 正确写法

```ts
import user from './test';

const { name, age } = user;
```

### 3. 错误理解

下面这种写法不是对默认导出对象做解构，而是在导入命名导出：

```ts
import { name, age } from './test';
```

这要求原文件必须有：

```ts
export const name = 'Tom';
export const age = 20;
```

所以要注意：

- `import { xxx }` 是命名导入
- `const { xxx } = obj` 才是对象解构

## 八、常见使用场景

### 1. 使用 export 的场景

当一个文件中需要导出多个内容时，通常使用 `export`。

例如工具函数文件：

```ts
export function add(a: number, b: number) {
  return a + b;
}

export function sub(a: number, b: number) {
  return a - b;
}
```

### 2. 使用 export default 的场景

当一个文件只需要导出一个核心内容时，通常使用 `export default`。

例如：

```ts
export default App;
export default request;
export default store;
```

------

## 九、React 项目中的常见写法

### 1. 页面组件默认导出

```tsx
function Home() {
  return <div>Home</div>;
}

export default Home;
```

导入：

```tsx
import Home from './pages/Home';
```

### 2. 工具函数命名导出

```ts
export function formatDate() {}
export function formatMoney() {}
```

导入：

```ts
import { formatDate, formatMoney } from './utils';
```

## 十、TypeScript 中的补充说明

在 TypeScript 中，类型也可以导出。

### 1. 导出类型

```ts
export type User = {
  name: string;
  age: number;
};
```

导入：

```ts
import type { User } from './types';
```

### 2. 默认导出类型较少见

虽然语法上可以写，但实际开发中更常见的是使用命名导出类型，因为可读性更好，也更方便统一管理。

------

## 十一、初学者常见问题

### 1. 为什么有时候导入要加 `{}`，有时候不要

原因在于导出方式不同：

- `export` 对应 `import { xxx }`
- `export default` 对应 `import xxx`

### 2. 为什么默认导入的名字可以改

因为默认导出只有一个，导入时模块系统知道你拿的是“默认那个内容”，所以名字可以自定义。

### 3. 为什么命名导入的名字一般不能乱写

因为命名导入是按名字匹配的，导出叫什么，导入时就要取什么名字。
如果想改名，要使用 `as`。

------

## 十二、总结

### 1. export

- 用于命名导出
- 可以导出多个内容
- 导入时需要 `{}``
- 名字一般要对应

### 2. export default

- 用于默认导出
- 一个模块通常只有一个
- 导入时不需要 `{}``
- 名字可以自定义

### 3. 使用建议

- 一个文件只导出一个核心内容时，优先考虑 `export default`
- 一个文件需要导出多个内容时，优先考虑 `export`
- 组件文件常用 `export default`
- 工具函数、类型、常量集合常用 `export`

------

## 十三、记忆方式

可以这样记：

- `export` 是“分别点名导出”
- `export default` 是“默认导出最重要的那个”

最常见的对应关系：

```ts
export const add = () => {};
import { add } from './utils';
export default App;
import App from './App';
```
