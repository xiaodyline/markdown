这道题是**前端高频面试题**，核心不是让你背网络全流程，而是考察你对：

**URL 解析 → DNS → TCP/TLS → HTTP 请求 → 服务器响应 → 浏览器渲染**

这条链路是否清楚。

# 当浏览器输入 baidu.com 会发生什么？

可以按这条主线回答：

```txt
URL 解析
→ DNS 域名解析
→ 建立 TCP 连接
→ 如果是 HTTPS，还要 TLS 握手
→ 浏览器发送 HTTP 请求
→ 服务器处理并返回响应
→ 浏览器解析 HTML
→ 加载 CSS、JS、图片等资源
→ 构建 DOM、CSSOM、Render Tree
→ Layout 布局
→ Paint 绘制
→ Composite 合成
→ 页面展示
```

------

## 1. URL 解析

用户输入：

```txt
baidu.com
```

浏览器会先判断它是：

```txt
搜索关键词
还是一个 URL 地址
```

因为 `baidu.com` 符合域名格式，所以浏览器会把它当作网址。

如果没有写协议，浏览器通常会补全：

```txt
http://baidu.com
```

或者根据 HSTS、浏览器策略、历史访问记录跳转到：

```txt
https://www.baidu.com
```

------

## 2. DNS 域名解析

浏览器不能直接访问 `baidu.com`，它需要先知道这个域名对应的服务器 IP。

所以会进行 DNS 查询：

```txt
baidu.com → IP 地址
```

DNS 查询大致会按缓存顺序查找：

```txt
浏览器缓存
→ 操作系统缓存
→ hosts 文件
→ 本地 DNS 服务器
→ 根域名服务器
→ 顶级域名服务器
→ 权威 DNS 服务器
```

最终得到类似这样的 IP 地址。

```txt
baidu.com → xxx.xxx.xxx.xxx
```

面试中可以简单说：

> 浏览器会先通过 DNS 把域名解析成 IP 地址，因为真正进行网络通信时需要的是 IP，而不是域名。

------

## 3. 建立 TCP 连接

拿到 IP 后，浏览器会和服务器建立 TCP 连接。

TCP 建立连接需要三次握手：

```txt
客户端：你能收到吗？
服务器：我能收到，你能收到吗？
客户端：我也能收到，开始通信。
```

更专业一点说：

```txt
SYN
SYN + ACK
ACK
```

三次握手的作用是：

```txt
确认客户端和服务器双方的发送、接收能力都正常
```

------

## 4. HTTPS 还会进行 TLS 握手

如果访问的是：

```txt
https://www.baidu.com
```

那么 TCP 连接之后，还要进行 TLS 握手。

TLS 的作用是：

```txt
加密通信
验证服务器身份
防止数据被窃听和篡改
```

大致过程是：

```txt
浏览器和服务器协商加密算法
服务器返回证书
浏览器验证证书是否合法
双方生成会话密钥
后续通信使用密钥加密
```

面试里不一定要讲太深，但可以补一句：

> 如果是 HTTPS，请求发送前还会先进行 TLS 握手，用来建立安全加密通道。

------

## 5. 发送 HTTP 请求

连接建立后，浏览器会发送 HTTP 请求。

请求大概长这样：

```http
GET / HTTP/1.1
Host: www.baidu.com
User-Agent: Chrome
Accept: text/html
Cookie: ...
```

核心信息包括：

```txt
请求方法：GET
请求路径：/
请求头：Host、User-Agent、Cookie 等
```

如果之前访问过百度，浏览器可能还会携带 Cookie。

------

## 6. 服务器处理请求并返回响应

百度服务器收到请求后，会根据请求路径、请求头、Cookie 等信息处理请求，然后返回响应。

响应大概包括：

```txt
状态码
响应头
响应体
```

例如：

```http
HTTP/1.1 200 OK
Content-Type: text/html
```

常见状态码：

```txt
200：请求成功
301 / 302：重定向
304：使用缓存
404：资源不存在
500：服务器错误
```

访问 `baidu.com` 时，也可能先返回重定向，例如跳转到：

```txt
https://www.baidu.com
```

------

## 7. 浏览器解析 HTML

浏览器拿到 HTML 后，开始解析。

首先构建：

```txt
DOM 树
```

DOM 树来自 HTML：

```html
<html>
  <head></head>
  <body></body>
</html>
```

如果 HTML 中遇到 CSS：

```html
<link rel="stylesheet" href="style.css">
```

浏览器会下载并解析 CSS，构建：

```txt
CSSOM 树
```

如果遇到 JS：

```html
<script src="main.js"></script>
```

浏览器会下载并执行 JS。

需要注意：

```txt
普通 script 会阻塞 HTML 解析
CSS 会影响渲染
图片不会阻塞 DOM 解析，但会影响页面加载完成时间
```

------

## 8. 构建渲染树 Render Tree

浏览器会把：

```txt
DOM 树 + CSSOM 树
```

合并成：

```txt
Render Tree 渲染树
```

Render Tree 中只包含需要显示的节点。

比如：

```css
display: none;
```

对应的元素不会进入渲染树。

------

## 9. 布局、绘制、合成

接下来浏览器进行渲染流程：

```txt
Layout 布局：计算元素的位置和大小
Paint 绘制：把文字、颜色、边框、阴影等画出来
Composite 合成：把不同图层合成最终页面
```

最终用户看到完整页面。

------

# 面试回答版本

可以这样答：

> 在浏览器输入 `baidu.com` 后，浏览器首先会解析输入内容，判断它是 URL，并补全协议。然后通过 DNS 将域名解析成服务器 IP 地址。拿到 IP 后，浏览器会和服务器建立 TCP 连接，如果是 HTTPS，还会进行 TLS 握手来建立安全通道。连接建立后，浏览器发送 HTTP 请求，服务器处理请求并返回 HTTP 响应，一般是 HTML文档。浏览器拿到 HTML 后开始解析，构建 DOM 树，解析 CSS 构建 CSSOM 树，然后合成 Render Tree，经过布局、绘制和合成，最终把页面展示出来。如果过程中涉及缓存、重定向、Cookie，也会参与请求和响应流程。

------

# 更简洁的记忆版

记住这 8 个词：

```txt
解析 → DNS → TCP → TLS → 请求 → 响应 → 解析 → 渲染
```

对应完整流程：

```txt
URL 解析
DNS 解析
TCP 三次握手
TLS 握手
发送 HTTP 请求
服务器返回 HTTP 响应
浏览器解析资源
页面渲染展示
```

你可以先试着用这 8 个词，自己口头复述一遍。
