#### 61. 从输入 URL 到页面展示全过程？

```
URL 解析
→ DNS 域名解析
→ 建立 TCP 连接
→ 如果是 HTTPS，还要 TLS 握手
→ 浏览器发送 HTTP 请求
→ 服务器处理并返回响应
→ 浏览器解析 HTML
→ 加载 CSS、JS、图片等资源
→ 构建 DOM、CSSOM、Render Tree
→ Layout 布局(回流reflow)
→ Paint 绘制（重绘）
→ Composite 合成
→ 页面展示
```

#### 62. TCP 三次握手过程？为什么是三次？

TCP 三次握手是客户端和服务器建立连接的过程，目的是确认双方的发送能力和接收能力都正常，并同步初始序列号。

1. 客户端发送 SYN 报文 + seq:x。
2. 服务端返回 SYN + ACK + ack:x+1 + seq:y。
3. 客户端发送 ACK + ack:y+1。
   三次握手的原因：**防止已失效的连接请求报文突然又传送到服务端，导致错误连接**。



#### 63. TCP 四次挥手过程？为什么是四次？

1. 客户端发送 FIN。
2. 服务端返回 ACK。
3. 服务端发送 FIN。
4. 客户端返回 ACK。
   原因：TCP 是全双工，每个方向都需要单独关闭。





#### 64. HTTP 和 HTTPS 区别？

- HTTP：明文传输，不安全。
- HTTPS：HTTP + SSL/TLS，加密传输，更安全。

HTTP 是超文本传输协议，数据默认是明文传输，安全性较低，默认端口是 80。HTTPS 可以理解为 HTTP 加上 TLS/SSL 安全层，默认端口是 443。HTTPS 会通过数字证书验证服务器身份，并对传输数据进行加密和完整性校验，可以防止数据被窃听、篡改和冒充。HTTPS 在建立连接时会多一次 TLS 握手和加解密开销，但现在已经是主流网站的标准配置。

#### 66. HTTP 1.1 和 HTTP 2.0 区别？

- HTTP 1.1：
  - 长连接（Keep-Alive）。
  - 管道化，但队头阻塞。
- HTTP 2.0：
  - 二进制分帧。
  - 多路复用。
  - 头部压缩。
  - 服务器推送+设置请求优先级。



#### 67. 什么是浏览器缓存？强缓存和协商缓存？

浏览器缓存是浏览器把请求过的资源保存到本地，下次访问时尽量复用本地资源，

- 强缓存：不请求服务器，直接从缓存读取。
  - `Cache-Control`、`Expires`。
- 协商缓存：向服务器确认缓存是否有效。
  - `Last-Modified` / `If-Modified-Since`。
  - `ETag` / `If-None-Match`。



#### 68. 什么是 Cookie、`SessionStorage`、`LocalStorage`？

它们都是浏览器端存储数据的方式，但作用和生命周期不同。

- Cookie：每次请求都会携带，大小有限。
- `LocalStorage`：持久化本地存储，除非主动删除。
- `SessionStorage`：会话级别，关闭标签页就清除。

Cookie、SessionStorage 和 LocalStorage 都是浏览器存储数据的方式。Cookie 容量较小，通常约 4KB，可以设置过期时间，并且会自动随同源请求发送，常用于登录态和服务端会话标识。SessionStorage 是会话级存储，刷新页面不会丢失，但关闭标签页后会清除，适合存临时页面状态。LocalStorage 是长期本地存储，关闭浏览器后仍然存在，适合存主题、语言偏好等非敏感数据。需要注意的是，SessionStorage 和 LocalStorage 不会自动随请求发送，而且都能被 JavaScript 读取，所以不适合存放敏感信息。

#### 75. 什么是 DNS 预解析？

提前解析域名，减少延迟。

```html
<link rel="dns-prefetch" href="//example.com">
```

DNS 预解析是浏览器性能优化手段之一，作用是提前把页面后续可能访问的域名解析成 IP 地址，减少真正请求资源时的 DNS 查询时间。通常使用 `<link rel="dns-prefetch" href="//cdn.example.com">`。它只解析域名，不会下载资源。和 `preload` 不同，`preload` 是提前加载具体资源；和 `preconnect` 不同，`preconnect` 不只做 DNS 解析，还会提前建立 TCP 连接和 TLS 握手。

#### 76. 什么是重排（回流）和重绘？

- 重排：布局变化，需要重新计算几何属性。
- 重绘：外观变化，不影响布局。
  重排必定重绘，重绘不一定重排。



#### 77. 如何减少重排重绘？

- 批量修改 DOM。
- 批量修改样式
- 避免频繁读取布局属性交叉。
- 让频繁变化的元素脱离文档流
- 动画尽量使用 `transform` 代替 `top/left`。
- 使用虚拟列表
- 使用 `visibility` 代替 `display: none`（有时）。



#### 78. 什么是虚拟 DOM？

虚拟 DOM 是用 JS 对象描述 DOM 结构，通过 diff 算法最小化更新，减少真实 DOM 操作。



#### 79. 什么是 SPA？优缺点？

SPA 是单页面应用，通常只有一个 HTML 页面，页面切换时不会重新请求新的 HTML，而是通过前端路由和 JavaScript 动态渲染对应组件。

优点：

- 用户体验好，页面流畅。

- 前后端分离。

缺点：

- SEO 不友好。

- 首屏加载可能慢。
