JavaScript 中 `...`、`arguments` 相关概念整理

1. 总体分类

JavaScript 里看到 `...`，不要直接统一叫“展开运算符”。
它有两类主要用法：

1. 展开（spread）
2. 收集剩余项（rest）

判断方式只看位置。

1. 一张总表

写法：`fn(...arr)`
位置：函数调用时
名称：展开
作用：把数组拆成多个参数

写法：`[...arr1, ...arr2]`
位置：数组字面量中
名称：展开
作用：把数组元素展开到新数组

写法：`{ ...obj1, ...obj2 }`
位置：对象字面量中
名称：展开
作用：把对象属性展开到新对象

写法：`function fn(...args) {}`
位置：函数参数位置
名称：剩余参数
作用：把多个参数收集成数组

写法：`const [a, ...rest] = arr`
位置：数组解构左侧
名称：剩余元素
作用：收集未被单独取出的剩余项

写法：`const { a, ...rest } = obj`
位置：对象解构左侧
名称：剩余元素
作用：收集剩余属性

写法：`arguments`
位置：普通函数内部
名称：arguments 对象
作用：保存传入函数的所有实参

1. 展开（spread）

3.1 定义

展开的本质是：
把一个整体拆开，放到另一个位置。

3.2 函数调用中的展开

```js
function add(a, b, c) {
  return a + b + c;
}

const nums = [1, 2, 3];

console.log(add(...nums)); // 6
```

等价理解：

```js
add(...nums)
```

相当于：

```js
add(1, 2, 3)
```

说明：

这里的 `...nums` 是展开。
不是解构，不是剩余参数。

3.3 数组中的展开

```js
const arr1 = [1, 2];
const arr2 = [3, 4];

const result = [...arr1, ...arr2];
console.log(result); // [1, 2, 3, 4]
```

常见用途：

1. 合并数组
2. 复制数组
3. 在数组中插入其他数组内容

3.4 复制数组

```js
const arr1 = [1, 2, 3];
const arr2 = [...arr1];

console.log(arr2); // [1, 2, 3]
console.log(arr1 === arr2); // false
```

说明：

这是浅拷贝，不是深拷贝。

3.5 对象中的展开

```js
const user = { name: "Tom", age: 20 };
const extra = { city: "Tokyo" };

const result = { ...user, ...extra };
console.log(result);
// { name: "Tom", age: 20, city: "Tokyo" }
```

3.6 对象属性覆盖

```js
const obj1 = { name: "Tom", age: 20 };
const obj2 = { age: 21 };

const result = { ...obj1, ...obj2 };
console.log(result);
// { name: "Tom", age: 21 }
```

说明：

后面的同名属性会覆盖前面的。

1. 剩余参数（rest parameters）

4.1 定义

当 `...变量名` 出现在函数参数位置时，表示：

把多出来的实参收集成一个数组。

```js
function fn(...args) {
  console.log(args);
}

fn(1, 2, 3); // [1, 2, 3]
```

结论：

这里的 `args` 是真数组。

4.2 基本案例

```js
function sum(...args) {
  return args.reduce((total, item) => total + item, 0);
}

console.log(sum(1, 2, 3)); // 6
console.log(sum(10, 20, 30, 40)); // 100
```

4.3 和普通参数一起使用

```js
function fn(a, b, ...rest) {
  console.log(a);    // 1
  console.log(b);    // 2
  console.log(rest); // [3, 4, 5]
}

fn(1, 2, 3, 4, 5);
```

4.4 规则

剩余参数必须写在最后：

```js
function fn(...args, a) {}
```

上面是错误写法。

1. 解构中的剩余元素（rest element）

5.1 定义

当 `...变量名` 出现在解构赋值左边时，它表示：

收集剩下没被单独取出的部分。

5.2 数组解构中的剩余元素

```js
const arr = [1, 2, 3, 4];

const [first, ...rest] = arr;

console.log(first); // 1
console.log(rest);  // [2, 3, 4]
```

说明：

这里不是展开，而是收集剩余项。

5.3 对象解构中的剩余元素

```js
const user = {
  name: "Tom",
  age: 20,
  city: "Tokyo"
};

const { name, ...rest } = user;

console.log(name); // "Tom"
console.log(rest); // { age: 20, city: "Tokyo" }
```

1. arguments

6.1 定义

`arguments` 是普通函数内部自带的一个对象，表示调用函数时传入的所有参数。

```js
function fn() {
  console.log(arguments);
}

fn(1, 2, 3);
```

输出效果类似：

```js
Arguments(3) [1, 2, 3]
```

6.2 arguments 的特点

1. 有下标
2. 有 `length`
3. 不是数组
4. 是类数组对象

```js
function fn() {
  console.log(arguments[0]); // 1
  console.log(arguments.length); // 3
  console.log(Array.isArray(arguments)); // false
}

fn(1, 2, 3);
```

6.3 arguments 不能直接使用数组方法

```js
function fn() {
  console.log(arguments.map(item => item * 2));
}

fn(1, 2, 3);
```

这会报错，因为 `arguments` 不是真数组。

6.4 把 arguments 转成数组

写法一：`Array.from`

```js
function fn() {
  const arr = Array.from(arguments);
  console.log(arr.map(item => item * 2));
}

fn(1, 2, 3); // [2, 4, 6]
```

写法二：展开成数组

```js
function fn() {
  const arr = [...arguments];
  console.log(arr);
}

fn(1, 2, 3); // [1, 2, 3]
```

1. `...args` 和 `arguments` 的区别

对比项：所在位置
`...args`：函数参数里
`arguments`：普通函数内部

对比项：本质
`...args`：剩余参数
`arguments`：内置类数组对象

对比项：是否真数组
`...args`：是
`arguments`：否

对比项：是否能直接用 `map/filter/reduce`
`...args`：能
`arguments`：不能

对比项：是否推荐
`...args`：推荐
`arguments`：旧写法，了解即可

对比项：箭头函数中能否使用
`...args`：能
`arguments`：没有自己的 `arguments`

对比例子：

`arguments` 写法

```js
function sum() {
  let total = 0;
  for (let i = 0; i < arguments.length; i++) {
    total += arguments[i];
  }
  return total;
}

console.log(sum(1, 2, 3)); // 6
```

`...args` 写法

```js
function sum(...args) {
  return args.reduce((total, item) => total + item, 0);
}

console.log(sum(1, 2, 3)); // 6
```

结论：

现代开发优先用 `...args`。

1. 箭头函数和 arguments

箭头函数没有自己的 `arguments`。

```js
const fn = () => {
  console.log(arguments);
};

fn(1, 2, 3);
```

这通常会报错，或者读到外层作用域的 `arguments`。

正确写法：

```js
const fn = (...args) => {
  console.log(args);
};

fn(1, 2, 3); // [1, 2, 3]
```

1. 最容易混淆的点

9.1 `...nums` 一定叫展开运算符吗

不一定，要看位置。

在函数调用里：

```js
add(...nums)
```

这是展开。

在函数参数里：

```js
function fn(...args) {}
```

这是剩余参数。

在解构左边：

```js
const [a, ...rest] = arr;
```

这是解构中的剩余元素。

9.2 `(...args)` 是不是解构

不是。

```js
function fn(...args) {}
```

这是剩余参数，不是解构。

解构一般长这样：

```js
const [a, b] = [1, 2];
const { name, age } = user;
```

9.3 `...` 到底是“展开”还是“收集”

判断口诀：

右边用，通常是展开。
左边用，通常是收集。

例如：

展开：

```js
fn(...arr);
[...arr1, ...arr2];
{ ...obj1, ...obj2 };
```

收集：

```js
function fn(...args) {}
const [a, ...rest] = arr;
const { a, ...rest } = obj;
```

1. 常用案例整理

10.1 合并数组

```js
const a = [1, 2];
const b = [3, 4];

const c = [...a, ...b];
console.log(c); // [1, 2, 3, 4]
```

10.2 合并对象配置

```js
const defaultConfig = {
  host: "localhost",
  port: 3000,
  debug: false
};

const userConfig = {
  port: 8080,
  debug: true
};

const finalConfig = { ...defaultConfig, ...userConfig };

console.log(finalConfig);
// { host: 'localhost', port: 8080, debug: true }
```

10.3 不定参数求和

```js
function sum(...args) {
  return args.reduce((a, b) => a + b, 0);
}

console.log(sum(1, 2, 3)); // 6
console.log(sum(10, 20, 30)); // 60
```

10.4 取数组前几个，剩余的收集起来

```js
const nums = [10, 20, 30, 40, 50];

const [first, second, ...others] = nums;

console.log(first);  // 10
console.log(second); // 20
console.log(others); // [30, 40, 50]
```

10.5 把 `arguments` 转成数组再过滤

```js
function fn() {
  const arr = Array.from(arguments);
  return arr.filter(item => item > 10);
}

console.log(fn(5, 12, 20, 8)); // [12, 20]
```

1. 常见错误

11.1 把 `arguments` 当数组直接用

```js
function fn() {
  return arguments.map(item => item * 2);
}
```

错误原因：`arguments` 不是真数组。

11.2 剩余参数没有写在最后

```js
function fn(...args, a) {}
```

错误原因：剩余参数必须放最后。

11.3 误以为展开是深拷贝

```js
const obj1 = {
  user: { name: "Tom" }
};

const obj2 = { ...obj1 };
obj2.user.name = "Jack";

console.log(obj1.user.name); // Jack
```

原因：展开只是浅拷贝。

1. 最终总结

12.1 看到 `...` 时怎么判断

先看位置：

1. 函数调用、数组字面量、对象字面量里
   是展开
2. 函数参数位置
   是剩余参数
3. 解构赋值左边
   是剩余元素

12.2 看到 `arguments` 时怎么判断

1. 只会出现在普通函数内部
2. 表示所有实参
3. 是类数组对象
4. 不是数组
5. 现代开发一般用 `...args` 替代

12.3 一页速记

展开：

```js
const arr = [1, 2, 3];
console.log(...arr);
```

剩余参数：

```js
function fn(...args) {
  console.log(args);
}
```

数组解构中的剩余元素：

```js
const [a, ...rest] = [1, 2, 3];
```

对象解构中的剩余元素：

```js
const { name, ...rest } = user;
```

arguments：

```js
function fn() {
  console.log(arguments);
}
```

arguments 转数组：

```js
const arr = Array.from(arguments);
const arr2 = [...arguments];
```
