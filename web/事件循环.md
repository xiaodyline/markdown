# 事件循环

## 一、事件循环是什么

事件循环是 JavaScript 运行时中的一种**任务调度机制**。

它的作用是：

- 让同步代码按顺序执行
- 让异步任务在条件满足后得到执行
- 协调调用栈、宿主环境、任务队列之间的运行顺序

可以直接记一句话：

**同步代码先执行，异步任务先等待，满足条件后进入任务队列，调用栈空了以后，事件循环先执行微任务，再执行宏任务。**

------

## 二、为什么需要事件循环

JavaScript 本身通常是**单线程**的。

这意味着同一时刻，主线程只能做一件事。
但实际开发中又需要处理很多任务，例如：

- 定时器
- 网络请求
- 用户点击
- Promise 回调

如果没有事件循环，这些任务就很难协调执行。

所以事件循环的本质就是：

**让单线程的 JavaScript 也能有序处理同步任务和异步任务。**

------

## 三、事件循环中的核心概念

### 1. 调用栈

调用栈是**当前正在执行代码的地方**。

谁在执行，谁就在调用栈中；执行完毕后，就从调用栈中移出。

调用栈负责的是：

- 执行同步代码
- 执行被调度回来的异步回调

------

### 2. 宿主环境

宿主环境指浏览器或 Node.js 这样的运行环境。

JavaScript 中很多异步能力并不是语言本身提供的，而是宿主环境提供的，例如：

- 定时器
- 事件监听
- 网络请求
- 文件读写

宿主环境负责的是：

- 等待异步条件成熟
- 条件成熟后，把对应任务放入任务队列

------

### 3. 任务队列

任务队列是**已经满足执行条件，但还没有真正执行的任务等待区**。

也就是说：

- 还没满足条件的异步任务，不在任务队列里
- 满足条件之后，才会进入任务队列排队

任务队列分为两类。

#### 3.1 微任务

微任务优先级更高。
它会在**当前这一轮任务结束后立即执行**。

常见微任务：

- Promise 的 then、catch、finally
- queueMicrotask
- MutationObserver

#### 3.2 宏任务

宏任务优先级低于微任务。
它通常在**下一轮事件循环中执行**。

常见宏任务：

- 整体脚本
- setTimeout
- setInterval
- DOM 事件回调
- I/O 回调

------

### 4. 事件循环

事件循环就是整个调度过程。

它会不断检查：

- 调用栈是否为空
- 微任务队列中是否有任务
- 宏任务队列中是否有任务

然后按照规则进行调度。

------

## 四、事件循环的执行流程

可以按下面顺序理解：

### 第一步

先执行调用栈中的同步代码。

### 第二步

遇到异步任务时，不直接执行，而是交给宿主环境处理。

### 第三步

当异步任务满足条件后，把对应回调放入任务队列。

### 第四步

当调用栈清空后，事件循环开始调度任务队列。

### 第五步

先执行**所有微任务**。

### 第六步

再执行**一个宏任务**。

### 第七步

这个宏任务执行完后，再次清空微任务队列。

### 第八步

继续下一轮循环。

------

## 五、最重要的执行规则

### 1. 同步代码一定先执行

只要还在调用栈中运行，同步代码就不会被异步回调打断。

### 2. 异步任务先等待，不会立刻执行

异步任务要先交给宿主环境，等条件成熟后，才有机会进入任务队列。

### 3. 微任务优先于宏任务

调用栈空了以后，不是先执行宏任务，而是先清空微任务。

### 4. 每执行完一个宏任务，都会再次清空微任务

这是事件循环中最关键的一条规则。

------

## 六、它们之间的关系

可以用这句话整体记忆：

**调用栈负责执行，宿主环境负责等待，任务队列负责排队，事件循环负责调度。**

再细一点是：

- 同步代码进入调用栈执行
- 异步任务交给宿主环境等待
- 条件成熟后进入任务队列
- 事件循环在调用栈空闲时进行调度
- 调度顺序是先微任务，后宏任务

------

## 七、容易混淆的点

### 1. 调用栈不等于同步代码区

更准确地说，调用栈是**当前正在执行代码的地方**。
同步代码会进入调用栈，异步回调真正开始执行时也会进入调用栈。

### 2. 任务队列里不是“还没满足条件的任务”

任务队列里放的是：

**已经满足执行条件，但还在等待调度的任务。**

### 3. 宏任务和微任务都属于任务队列

只是它们优先级不同：

- 微任务优先级高
- 宏任务优先级低

### 4. 满足条件不等于立刻执行

异步任务满足条件后，只是说明它可以进入任务队列了。
真正什么时候执行，还要看调用栈是否空闲，以及事件循环的调度顺序。

------

## 八、最简流程图

```text
同步代码进入调用栈执行
        ↓
遇到异步任务，交给宿主环境
        ↓
异步条件成熟
        ↓
进入任务队列
        ↓
调用栈清空
        ↓
先执行所有微任务
        ↓
再执行一个宏任务
        ↓
再清空微任务
        ↓
进入下一轮循环
```

------

## 九、最终总结

### 事件循环定义

事件循环是 JavaScript 中协调同步任务与异步任务执行顺序的一种调度机制。

### 一句话总结

**同步代码先执行，异步任务先等待，满足条件后进入任务队列，调用栈空了以后，先执行微任务，再执行宏任务。**

### 核心记忆版

- 调用栈：负责执行
- 宿主环境：负责等待
- 任务队列：负责排队
- 事件循环：负责调度
- 调度顺序：**先微任务，后宏任务**

------

## 十、适合背诵的简版

**JavaScript 的事件循环是指，主线程先执行调用栈中的同步代码；异步任务交给宿主环境等待；当条件满足后，对应回调进入任务队列；调用栈清空后，事件循环先执行所有微任务，再执行一个宏任务，并不断重复这一过程。**
