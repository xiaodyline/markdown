#### 7. Webpack 核心概念有哪些？

- `entry`：入口。
- `output`：输出。
- `moulde.loader`：处理非 JS 文件。
- `plugin`：扩展功能。
- `mode`：开发/生产模式。



#### 8. Loader 和 Plugin 区别？

- Loader：转换文件内容，如 `babel-loader`、`css-loader`。
- Plugin：扩展构建流程，如 `HtmlWebpackPlugin`、`DefinePlugin`。



#### 9. Webpack 构建流程？

1. 初始化参数。
2. 创建 Compiler 对象。
3. 加载所有插件。
4. 根据 entry 找到入口文件。
5. 编译模块，递归处理依赖。
6. 输出资源到 output 目录。



#### 10. 什么是 Tree Shaking？

Tree Shaking 是在打包时删除未使用的代码，依赖 ES6 模块静态分析。



#### 11. Vite 为什么比 Webpack 快？

- Vite 利用浏览器原生 ES 模块，开发时不打包，编译按需进行。
- Webpack 需要先打包再启动开发服务器。



#### 12. 什么是 Git Flow？

Git Flow 是一种分支模型：

- `master`：生产分支。
- `develop`：开发分支。
- `feature`：功能分支。
- `release`：发布分支。
- `hotfix`：紧急修复分支。

#### 13. `git rebase` 和 `git merge` 区别？

- `merge`：保留分支历史，创建一个合并提交。
- `rebase`：将当前分支移到目标分支顶端，保持线性历史。

#### 14.`git pull` 和 `git fetch` 区别？

- `git fetch`：只拉取远程更新，不合并。
- `git pull`：`fetch + merge`，拉取并合并。
